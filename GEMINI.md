# 프로젝트 개발 지침 (GEMINI.md)

홍익대학교 세종캠퍼스 3D 맵 프로젝트의 개발 컨벤션 및 관리 규칙입니다.

## 🚫 보안 및 관리 규칙
1. **Git 조작 금지**: 에이전트는 어떠한 경우에도 `git add`, `git commit`, `git push` 등 소스 제어 명령을 직접 수행하지 않습니다. 모든 변경 사항은 파일 수정까지만 완료하며, 커밋 및 배포 조작은 사용자가 직접 수행합니다.
2. **자격 증명 보호**: .env, API 키 등 민감한 정보는 절대 노출하거나 수정하지 않습니다.

## 📌 개발 원칙
1. **일관된 시각 효과**: 모든 층 전환 애니메이션 속도는 `0.02~0.03` 사이의 보간(Lerp) 값을 사용하여 부드러움을 유지합니다.
2. **데이터 중심 관리**: 강의실 정보 및 좌표는 `main.js` 및 `data/*.js` 파일을 통해 관리하며, 핀 높이는 층 바닥 기준의 로컬 좌표(Relative Height) 방식을 사용합니다.
3. **사용자 경험(UX)**: 
    - 강의실 선택 시 해당 층으로 자동 전환(Crop) 및 카메라 포커싱 기능을 제공합니다.
    - 층 전환 시 이전 선택된 건물 정보 및 핀 하이라이트를 자동으로 초기화합니다.
    - 자동 카메라 이동 중 사용자가 조작을 시작하면 즉시 자동 이동을 중단합니다.

## 🧊 3D 모델 및 리소스 관리
- **경로**: `/public/models/` 내에 저장하며 소스 코드에서는 절대 경로(`/models/...`)를 사용합니다.
- **단위**: 
    - 일반 건물: 층별 `65px` 간격 배치.
    - **A/B/C/D동**: `32.5px`, **E동**: `25px` 간격으로 조정하여 밀착감 최적화.
    - **C/D동 층 표기 매핑**: 데이터상의 1~6층을 UI상에서 **B1, 1층, 2층, 3층, 4층, 5층**으로 변환 표기.
- **핀 디자인**: 슬림 디자인 적용 (다리 22.5px, 구체 2.6). 클릭 시 노란색 강조 효과 제공.
- **재질**: 모든 건물의 재질을 **순백색(`0xffffff`)**으로 통일하고 배경색과 대비시켜 가시성 확보.

## 🌐 정보 연동 및 배포
- **배포 (Cloudflare Pages)**: `vite.config.js`를 통해 멀티 페이지 빌드 지원.
- **모바일 최적화**: 
    - **바텀 시트(Bottom Sheet)**: 홈 화면에서 제스처 기반의 위아래 드래그 가능한 정보창 구현.
    - **통합 내비게이션**: 햄버거 버튼(☰)을 통한 전 페이지 공통 전체화면 메뉴 오버레이 적용.
    - **스크롤 최적화**: 서브 페이지별 독립 스크롤 및 상단바 가림 현상 정밀 보정 (120px 여백 확보).
