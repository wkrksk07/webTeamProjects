# 프로젝트 개발 지침 (GEMINI.md)

홍익대학교 세종캠퍼스 3D 맵 프로젝트의 개발 컨벤션 및 관리 규칙입니다.

## 📌 개발 원칙
1. **일관된 시각 효과**: 모든 층 전환 애니메이션 속도는 `0.02~0.03` 사이의 보간(Lerp) 값을 사용하여 부드러움을 유지합니다.
2. **데이터 중심 관리**: 강의실 정보 및 좌표는 `main.js` 내의 `roomData` 및 `floorConfigs` 객체를 통해 관리하며, 향후 JSON 파일로 분리할 예정입니다.
3. **사용자 경험(UX)**: 자동 카메라 이동 중 사용자가 조작을 시작하면 즉시 자동 이동을 중단하여 제어권을 사용자에게 돌려줍니다.

## 🧊 3D 모델 및 리소스 관리
- **경로**: `/public/models/` 내에 저장하며 소스 코드에서는 절대 경로(`/models/...`)를 사용합니다.
- **단위**: FBX 모델의 높이값(offset)은 층별 밀착을 위해 현재 `65px` 간격으로 배치합니다.
- **핀 디자인**: 얇은 실린더 다리(0.3) + 빨간색 구체(3.5) 디자인을 유지합니다. 선택 시에는 구체 색상을 노란색으로 변경합니다.

## 🌐 정보 연동 (Crawling)
- **식단 정보**: 홍익대학교 학식 메뉴 페이지를 대상으로 하며, `facility.html` 내에서 자동 로드 및 수동 갱신 기능을 지원합니다.
- **주의**: 학교 서버 부하 방지를 위해 실제 서비스 시에는 백엔드 캐싱 처리가 권장됩니다.

## 🗓 향후 로드맵 (Next Steps)
- [ ] **데이터 분리**: 강의실 좌표 및 정보를 외부 JSON 파일로 마이그레이션.
- [ ] **핀 직접 클릭 연동**: Raycasting을 사용하여 화면상의 핀을 직접 클릭했을 때 정보 표시.
- [ ] **React 마이그레이션**: UI 확장성 및 상태 관리를 위해 React + R3F 환경으로 전환 검토.
- [ ] **검색 기능**: 특정 강의실 이름을 검색하면 해당 위치로 즉시 이동하는 기능 추가.
