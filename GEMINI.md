# 프로젝트 개발 지침 (GEMINI.md)

홍익대학교 세종캠퍼스 3D 맵 프로젝트의 개발 컨벤션 및 관리 규칙입니다.

## 🚫 보안 및 관리 규칙
1. **Git 조작 금지**: 에이전트는 어떠한 경우에도 `git add`, `git commit`, `git push` 등 소스 제어 명령을 직접 수행하지 않습니다. 모든 변경 사항은 파일 수정까지만 완료하며, 커밋 및 배포 조작은 사용자가 직접 수행합니다.
2. **자격 증명 보호**: .env, API 키 등 민감한 정보는 절대 노출하거나 수정하지 않습니다.

## 📌 개발 원칙
1. **일관된 시각 효과**: 모든 층 전환 애니메이션 속도는 `0.02~0.03` 사이의 보간(Lerp) 값을 사용하여 부드러움을 유지합니다.
2. **데이터 중심 관리**: 강의실 정보 및 좌표는 `main.js` 내의 `roomData` 및 `floorConfigs` 객체를 통해 관리하며, 향후 JSON 파일로 분리할 예정입니다.
3. **사용자 경험(UX)**: 
    - 강의실 선택 시 해당 층으로 자동 전환(Crop) 및 카메라 포커싱 기능을 제공합니다.
    - 층 전환(All/2F/3F/4F) 시 이전 선택된 건물 정보 및 핀 하이라이트를 자동으로 초기화합니다.
    - 자동 카메라 이동 중 사용자가 조작을 시작하면 즉시 자동 이동을 중단하여 제어권을 사용자에게 돌려줍니다.

## 🧊 3D 모델 및 리소스 관리
- **경로**: `/public/models/` 내에 저장하며 소스 코드에서는 절대 경로(`/models/...`)를 사용합니다.
- **단위**: 
    - 일반 건물: 층별 `65px` 간격 배치.
    - **A/B/C/D동 특화**: 적층 효과 및 밀착감을 위해 `32.5px` 간격으로 조정. (B/C/D동 6층, A동 4층 구성)
    - **C/D동 층 표기 매핑**: 데이터상의 1~6층을 UI상에서 **B1, 1층, 2층, 3층, 4층, 5층**으로 변환하여 표기 (사용자 요청 반영).
    - **스케일 조정**: B동은 `0.5`, C동은 `0.25`로 축소하여 주변 건물과의 비율 최적화.
- **좌표**: `main.js` 내의 `buildingConfigs`를 통해 각 동의 중심 좌표와 회전 각도를 관리합니다. (B/C동 정밀 좌표 반영 완료, A/C/D동 180도 회전 보정 적용, D동 신규 모델 및 B1~5층 확장 반영 완료)
- **핀 디자인**: 25% 축소된 슬림 디자인 적용 (다리 22.5px, 구체 2.6). 층간 간섭 방지 및 시각적 세밀함 확보.
- **재질**: 모든 건물(A~I동)의 재질을 **순백색(`0xffffff`)**으로 통일하고 그림자 효과(`PCFSoftShadowMap`)를 강화하여 입체감 구현.

## 🌐 정보 연동 및 배포
- **배포 (Cloudflare Pages)**: 
    - `vite.config.js`를 통해 `index.html` 및 `facility.html` 멀티 페이지 빌드를 지원합니다.
    - 대시보드 빌드 설정: 빌드 명령 `npm run build`, 배포 명령 `exit 0`, 출력 디렉터리 `dist`.
- **식단 정보**: 홍익대학교 학식 메뉴 페이지를 대상으로 하며, `facility.html` 내에서 자동 로드 및 수동 갱신 기능을 지원합니다.

## 🛠 주요 기술 사양 (Update 2026-05-20)
- **3D 렌더링 엔진**: Three.js 기반의 `ExtrudeGeometry` 모델링 기술 도입. (Box 형태에서 Vertices 기반 정밀 셰이프로 업그레이드)
- **서버리스 아키텍처**: Cloudflare Pages Functions를 활용한 Edge 컴퓨팅 구현.
- **고도화된 캐싱 (Quad-Caching)**:
    - **Client**: LocalStorage & HTTP Cache
    - **Edge**: Cloudflare CDN Cache & KV Storage (이중 방어)
- **데이터 정합성**: 
    - `buildingConfigs`: 전 건물의 외곽선 좌표(Vertices) 및 높이 데이터 관리.
    - `isNumeric` 예외 처리: 호수(Room Number) 파싱 로직 정교화.
- **UX/UI 최적화**: 
    - 카카오 맵 API 연동을 통한 셔틀 정류장 자동 포커싱.
    - 공지사항 트리형 히스토리 뷰 및 독립 스크롤바 구현.
- **배포 인프라**: Cloudflare Pages (Clean URL 및 _redirects 최적화).
표 반영. C102(id: 02)호는 별도 요청 좌표(`58.43, 263.79, -237.92`)로 고도화.
    - **핀 동기화**: C동은 1층의 핀 데이터를 2~6층 전체에 동일하게 복사하여 전 층 시설 정보 일관성 확보.
    - `isNumeric` 예외 처리: '계단-1' 등 하이픈이 포함된 시설 명칭이 호수(Room Number)로 오인되지 않도록 정규 로직 강화.
- **건물 확장**: 
    - B동: 6층 모델 적층 완료 (`32.5px` 간격 유지). 2~6층은 핀 없이 모델만 노출하여 시각적 완성도 향상.
    - C동: 6층 모델 적층 및 전 층 핀 노출 완료 (`32.5px` 간격, 0.25 스케일 적용).
- **클라이언트 사이드 크롤링**:
    - **Proxy**: `allorigins.win`을 통해 브라우저 직접 크롤링 구현.
    - **Caching**: `localStorage`에 `savedDate`와 함께 데이터를 저장하여 하루 한 번만 서버 요청 수행 (효율성 극대화).
- **UI/UX**: 사이드바 렌더링 시 데이터 정의 순서를 보존하여 관리자가 의도한 정렬 상태 유지.
